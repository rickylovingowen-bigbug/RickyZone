# 习惯打卡产品 PRD

## 产品概述

一个支持 A/B 两类习惯追踪的打卡应用，帮助用户建立和维持良好习惯。
 
---
 
## 习惯类型定义

### A类习惯 - 每周进行
- **定义**：每周需要完成一定次数的重复习惯
- **示例**：每周运动3次、每周阅读5次
- **用户输入**：
  - 习惯名称
  - 每周目标次数（1-7次）

### B类习惯 - 累积进行
- **定义**：在可预见的未来总共要完成的目标
- **示例**：累计跑步100公里、读完10本书
- **用户输入**：
  - 习惯名称
  - 累积总目标次数
  - 截止日期（可选）

## 功能模块

### 1. 新建习惯

**界面元素：**
- 习惯名称输入框
- 习惯类型选择（A类/B类）
- 动态输入区域：
  - A类：每周目标次数（数字输入，1-7）
  - B类：累积总目标次数 + 截止日期选择器
- 颜色选择器

**业务规则：**
- 名称必填，最多50字符
- A类每周目标次数范围 1-7
- B类截止日期可选，到期自动归档

---

### 2. 周视图

**布局：**
- 顶部：周导航（上一周/下一周）
- 表格：习惯名称 + 周一至周日 + 统计列

**功能：**
- 每个习惯每天显示可点击的打卡按钮
- 打卡状态三选一：
  - ✓ 完成（绿色）
  - ✗ 未完成（红色）
  - ○ 未打卡（灰色空心）

**A类习惯统计列显示：**
- 本周完成率（当周完成次数 ÷ 每周目标次数）
- 历史累积完成次数

**B类习惯统计列显示：**
- 累积完成次数 / 目标总次数
- 完成比例百分比

---

### 3. 月视图

**布局：**
- 顶部：月份导航
- 日历网格：6周 × 7天

**显示内容：**
- 每天显示数字：当天完成的任务总数（所有习惯合计）
- 颜色深浅表示完成数量多少

**示例：**
- 0个任务：浅灰色
- 1个任务：浅绿色
- 2个任务：中绿色
- 3个及以上：深绿色

---

### 4. 日程视图（习惯列表）

**习惯卡片展示：**

**A类习惯卡片：**
- 习惯名称 + 类型标签
- 累积达成周数（从创建到现在，每周完成率100%的周数）
- 历史累积完成次数

**B类习惯卡片：**
- 习惯名称 + 类型标签
- 进度条（完成比例）
- 已完成次数 / 目标次数
- 截止日期（如有）

**操作按钮：**
- 暂停/恢复
- 归档
- 删除

## 数据结构

```typescript
// 习惯类型
type HabitType = 'A' | 'B';
type HabitStatus = 'active' | 'archived' | 'paused' | 'deleted';

// 习惯
interface Habit {
  id: string;
  name: string;
  color: string;
  habitType: HabitType;           // A或B
  weeklyTarget?: number;          // A类：每周目标次数
  totalTarget?: number;           // B类：累积总目标次数
  deadline?: string;              // B类：截止日期
  createdAt: string;
  updatedAt: string;
  status: HabitStatus;
}

// 打卡记录
interface CheckIn {
  id: string;
  habitId: string;
  date: string;
  status: 'completed' | 'failed' | 'pending';
  updatedAt: string;
}
```

## 核心算法

### 1. 本周完成率（A类）
```
本周完成率 = 本周完成次数 ÷ 每周目标次数 × 100%
```

### 2. 累积达成周数（A类）
```
从习惯创建日开始，遍历每一周：
- 如果该周完成率 ≥ 100%，累积周数 +1
- 返回总累积周数
```

### 3. B类进度
```
完成比例 = 累积完成次数 ÷ 目标总次数 × 100%
```

### 4. 自动归档
```
每天检查：
- 如果是B类习惯
- 如果有设置截止日期
- 如果截止日期 < 今天
- 自动将习惯状态改为"已归档"
```
