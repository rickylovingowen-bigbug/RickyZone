## 1. 产品概述

### 1.1 产品定位
一个面向小说创作者的**人物管理工具**，帮助作者系统化地记录、管理和梳理小说中的角色信息，特别是武侠/玄幻题材的人物关系网络。

### 1.2 整体架构

```
┌─────────────────────────────────────────┐
│              门户首页                    │
│  ┌─────────────┐    ┌─────────────┐    │
│  │  Habit      │    │   小说人物   │    │
│  │  Tracker    │    │   管理系统   │    │
│  │   [图标]    │    │   [图标]    │    │
│  └─────────────┘    └─────────────┘    │
└─────────────────────────────────────────┘
```

**系统入口**: 用户进入网站后，首先看到项目选择门户，点击对应图标进入各子系统。

### 1.3 用户价值
- **角色信息集中管理**: 避免人物设定散落在各处文档
- **快速检索筛选**: 按武学等级、门派、年龄等多维度查找角色
- **批量操作支持**: Excel 导入导出，方便数据迁移和备份
- **视觉化展示**: 武学等级颜色区分，一目了然

## 4. 数据结构

### 4.1 TypeScript 类型定义

```typescript
// 武学等级枚举
type MartialLevel = 'S' | 'A+' | 'A' | 'A-' | 'B+' | 'B' | 'B-' | 'C+' | 'C' | 'C-' | 'D';

// 性别枚举
type Gender = 'male' | 'female';

// 人物接口
interface Character {
  id: string;                    // 唯一标识
  name: string;                  // 姓名
  gender?: Gender;               // 性别
  age?: number;                  // 年龄
  sect?: string;                 // 门派
  sectPosition?: string;         // 门派职位
  faction?: string;              // 所属势力
  factionPosition?: string;      // 势力职位
  weapon?: string;               // 兵器
  martialLevel?: MartialLevel;   // 武学等级
  appearance?: string;           // 样貌
  personality?: string;          // 性格核心
  value?: string;                // 存在价值
  conflict?: string;             // 主要冲突
  createdAt: string;             // 创建时间 (ISO 8601)
  updatedAt: string;             // 更新时间 (ISO 8601)
}

// 筛选条件接口
interface CharacterFilter {
  martialLevels?: MartialLevel[];  // 武学等级（多选）
  sect?: string;                   // 门派
  ageSort?: 'asc' | 'desc';        // 年龄排序: asc正序, desc倒序
}

// 导入结果接口
interface ImportResult {
  success: number;           // 成功导入数量
  failed: number;            // 失败数量
  renamed: {                 // 重名自动重命名记录
    original: string;
    renamed: string;
  }[];
  errors: string[];          // 错误信息
}
```

### 4.2 本地存储设计

使用 IndexedDB (Dexie) 存储:

```typescript
// 数据库定义
const db = new Dexie('NovelCharacterDB');

db.version(1).stores({
  characters: '++id, name, sect, martialLevel, age, createdAt, updatedAt'
});
```

**索引说明**:
- `name`: 用于重名校验和搜索
- `sect`: 用于门派筛选
- `martialLevel`: 用于等级筛选和排序
- `age`: 用于年龄排序

### 6.3 关键算法

#### 6.3.1 人物排序算法

```typescript
function sortCharacters(characters: Character[]): Character[] {
  const levelOrder = ['S', 'A+', 'A', 'A-', 'B+', 'B', 'B-', 'C+', 'C', 'C-', 'D'];
  
  return [...characters].sort((a, b) => {
    // 1. 按武学等级排序（高到低）
    const levelDiff = levelOrder.indexOf(a.martialLevel || 'D') - 
                      levelOrder.indexOf(b.martialLevel || 'D');
    if (levelDiff !== 0) return levelDiff;
    
    // 2. 武学等级相同，按年龄倒序（大到小）
    return (b.age || 0) - (a.age || 0);
  });
}
```

#### 6.3.2 重名处理算法

```typescript
function generateUniqueName(existingNames: Set<string>, baseName: string): string {
  if (!existingNames.has(baseName)) return baseName;
  
  let counter = 1;
  let newName = `${baseName}${counter}`;
  
  while (existingNames.has(newName)) {
    counter++;
    newName = `${baseName}${counter}`;
  }
  
  return newName;
}
```
