### 4.2 人物数据模型（v2.6 更新）

#### 4.2.1 属性定义（v2.6 更新）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识 |
| name | string | 是 | 人物姓名 |
| gender | enum | 否 | 性别: male/female |
| age | number | 否 | 年龄 |
| sect | string | 否 | 所属门派 |
| sectPosition | string | 否 | 门派职位 |
| faction | string | 否 | 所属势力 |
| factionPosition | string | 否 | 势力职位 |
| weapon | string | 否 | 兵器 |
| martialLevel | enum | 否 | 武学 Rank（S+/S/S-/A+/A/A-/B+/B/B-/C+/C/C-/D+/D/D-）|
| martialTier | enum | 否 | 武学品级（v2.6 新增，自动根据 Rank 计算）|
| appearance | text | 否 | 样貌描述 |
| personality | text | 否 | 性格核心 |
| value | text | 否 | 存在价值 |
| conflict | text | 否 | 主要冲突 |

#### 4.2.2 武学体系（v2.6 更新）

**Rank 与品级对应关系（自动计算，不可编辑）**：

| 武学品级 | 对应 Rank 范围 | HEX颜色 |
|---------|---------------|---------|
| **无疆** | S+, S, S- | #7C3AED |
| **暮海** | A+, A, A- | #DC2626 |
| **阔原** | B+, B, B- | #F97316 |
| **长岭** | C+, C, C- | #3B82F6 |
| **清岫** | D+, D, D- | #9CA3AF |

**自动计算规则**：
- 用户选择/修改 Rank 时，系统自动计算并保存对应的武学品级
- 武学品级字段在编辑界面只读显示，不允许用户直接修改

#### 4.3.9 一键删除全部（v2.6 新增）

**按钮位置**：人物管理界面右上角，「导入」和「新建人物」按钮之间

**按钮样式**：
- 图标：垃圾桶图标
- 颜色：danger（红色系）
- 文案：简洁图标即可，或「全部删除」

**删除流程**：
1. 用户点击「全部删除」按钮
2. **第一次确认 Toast**：「确定要删除所有人物吗？此操作不可恢复。」
   - 选项：「取消」/「继续」
3. 用户点击「继续」
4. **第二次确认 Toast**：「再次确认：将删除全部 {N} 个人物，确定吗？」
   - 显示当前人物总数 N
   - 选项：「取消」/「确认删除」
5. 用户点击「确认删除」
6. 执行删除：清空 `db.characters` 表
7. 显示结果：「已删除全部 {N} 个人物」

**安全机制**：
- 必须连续两次确认才能删除
- 删除后数据不可恢复

#### 4.3.5 人物卡片视觉（v2.6 更新）

##### 4.3.5.1 卡片尺寸
- 保持竖版扑克牌式：160px × 240px（2:3 比例）

##### 4.3.5.2 信息展示顺序（v2.6 更新）

从上到下依次为：

| 顺序 | 信息项 | 显示规则 |
|------|--------|---------|
| 1 | 武学品级 | 如为空，显示「武学品级：未知」 |
| 2 | 武学 Rank | 如为空，显示「武学 Rank：未知」 |
| 3 | 所属门派 | 如为空，显示「所属门派：未知」 |
| 4 | 性别 | 如为空，显示「性别：未知」 |
| 5 | 年龄 | 如为空，显示「年龄：未知」 |
| 6 | 职位 | 如为空，显示「职位：未知」 |

##### 4.3.5.3 信息布局（v2.6 更新）

**布局方式**：
- 信息名称和实际信息距离靠近
- 不使用两端对齐（避免间隔过远）
- 采用紧凑布局：

```
武学品级：无疆
武学 Rank：S+
所属门派：武当
性别：男
年龄：25
职位：掌门
```

**样式规则**：
- 信息名称：text-text-tertiary（灰色）
- 实际信息：text-text-primary（主色），紧跟在冒号后
- 行间距紧凑

##### 4.3.5.4 顶部标签区域

保持现有设计：
- 左侧：武学 Rank 标签（带 Rank 配色）
- 右侧：所属门派标签（带门派配色）
