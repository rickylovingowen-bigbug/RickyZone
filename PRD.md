## v2.1 架构总览

### 2.1 产品定位
一个**多系统门户应用**，整合个人效率工具与创作辅助工具，并增加用户认证机制保护数据隐私：

## 第一部分：登录认证系统（v2.1 新增）

### 1.1 功能概述
为保护用户数据隐私，系统增加轻量级登录认证机制。用户必须登录后才能访问应用功能。

### 1.2 登录凭证
- **用户名**: `lovingowen`
- **密码**: `dawangbaxiaoni7`

> 注：当前版本采用硬编码凭证的简单认证方式，后续版本可扩展为多用户支持或后端认证。

### 1.3 登录流程

#### 1.3.1 登录页面 (LoginScreen)
- **页面路径**: `/login` (应用入口)
- **页面布局**: 居中卡片式设计
- **视觉风格**: 深色主题，与主应用保持一致

#### 1.3.2 登录表单元素

| 元素 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 用户名输入框 | text | 是 | 占位符："请输入用户名" |
| 密码输入框 | password | 是 | 占位符："请输入密码" |
| 登录按钮 | submit | - | 显示 LogIn 图标 + "登录" 文字 |
| 错误提示 | text | - | 验证失败时显示红色错误信息 |

#### 1.3.3 表单验证规则

1. **非空校验**
   - 用户名或密码为空时提示："请输入用户名和密码"

2. **凭证校验**
   - 用户名或密码不匹配时提示："用户名或密码错误"
   - 匹配成功后触发 `onLoginSuccess` 回调

3. **错误清除**
   - 用户在任一输入框输入时自动清除错误提示

### 1.4 会话管理

#### 1.4.1 登录状态存储
- **存储方式**: `localStorage`
- **存储字段**:
  - `isAuthenticated`: `'true'` (字符串)
  - `username`: 登录用户名

#### 1.4.2 登录状态流转
```
未登录 → [输入凭证] → 验证通过 → 存储状态 → 进入应用
   ↑                                              ↓
   └────────── [点击退出] ←───────────────────────┘
```

#### 1.4.3 自动登录
- 应用启动时检查 `localStorage.isAuthenticated`
- 如果为 `'true'`，自动恢复登录状态，跳过登录页
- 同时恢复 `username` 显示在页面头部

### 1.5 退出登录

#### 1.5.1 退出入口
- **位置**: 应用头部右上角（设置按钮旁）
- **图标**: `LogOut` (来自 lucide-react)
- **提示**: 鼠标悬停显示 "退出登录" Tooltip

#### 1.5.2 退出行为
- 清除 `localStorage` 中的 `isAuthenticated` 和 `username`
- 重置组件状态 `isAuthenticated = false`
- 页面自动跳转回登录页

### 1.6 登录后界面

#### 1.6.1 头部用户信息
- **位置**: 应用 Header 右侧
- **显示内容**: `欢迎，{username}`
- **响应式**: 小屏幕隐藏用户名，仅显示图标

#### 1.6.2 登录状态保护
- 未登录状态下访问任何页面自动重定向到登录页
- 登录成功后才能访问 Habit Tracker 和人物管理系统

## 第二部分：Habit Tracker（v1.0 核心功能）

> 该部分在 v2.1 中保持不变，作为子系统之一

### 2.1 习惯类型定义

#### A类习惯 - 每周进行
- **定义**: 每周需要完成一定次数的重复习惯
- **示例**: 每周运动3次、每周阅读5次
- **用户输入**: 习惯名称、每周目标次数（1-7次）

#### B类习惯 - 累积进行
- **定义**: 在可预见的未来总共要完成的目标
- **示例**: 累计跑步100公里、读完10本书
- **用户输入**: 习惯名称、累积总目标次数、截止日期（可选）

### 2.2 功能模块

#### 2.2.1 新建习惯
- 习惯名称输入框（必填，最多50字符）
- 习惯类型选择（A类/B类）
- 动态输入区域：
  - A类：每周目标次数（1-7）
  - B类：累积总目标次数 + 截止日期选择器
- 颜色选择器

#### 2.2.2 周视图
- 顶部周导航（上一周/下一周）
- 表格：习惯名称 + 周一至周日 + 统计列
- 打卡状态：✓完成（绿）、✗未完成（红）、○未打卡（灰）

#### 2.2.3 月视图
- 顶部月份导航
- 日历网格：6周 × 7天
- 热力图显示：每天完成任务总数（颜色深浅表示数量）

#### 2.2.4 日程视图（习惯列表）
- A类卡片：名称、累积达成周数、历史完成次数
- B类卡片：名称、进度条、完成比例、截止日期
- 操作：暂停/恢复、归档、删除

### 2.3 Habit Tracker 数据结构

```typescript
type HabitType = 'A' | 'B';
type HabitStatus = 'active' | 'archived' | 'paused' | 'deleted';

interface Habit {
  id: string;
  name: string;
  color: string;
  habitType: HabitType;
  weeklyTarget?: number;      // A类
  totalTarget?: number;       // B类
  deadline?: string;          // B类
  createdAt: string;
  updatedAt: string;
  status: HabitStatus;
}

interface CheckIn {
  id: string;
  habitId: string;
  date: string;
  status: 'completed' | 'failed' | 'pending';
  updatedAt: string;
}
```

### 2.4 Habit Tracker 颜色系统
- 主色调：蓝色 (#3B82F6)
- 成功/完成：绿色 (#10B981)
- 失败/未完成：红色 (#EF4444)
- 警告/暂停：橙色 (#F59E0B)

- **顶部导航**: 设置按钮、退出登录按钮

**人物卡片展示**：

**默认排序**: 武学等级（高→低）→ 年龄（大→小）

**筛选功能**（通过顶部筛选按钮触发）：
- 武学等级（多选）

| | 年龄 | 数字输入 | 范围 0-200 |

| | 势力职位 | 单行文本输入 | 如：盟主、堂主、护法等 |

- 默认高度：100px（约5行）
- 最大高度：200px（约10行）
- 支持自动扩展
- 字符限制：每个字段最多 500 字符
- 支持换行输入

**业务规则**:
- 仅「姓名」必填，其他可选
- 创建时校验姓名唯一性
- 重名提示错误，禁止保存
- 武学等级与 Rank 需对应（如不对应给出警告但允许保存）

#### 3.3.3 人物详情页
- 展示全部字段信息
- 长文本字段（样貌、性格核心、存在价值、主要冲突方向）以展开形式展示，支持滚动查看
- 操作：编辑（需确认Toast）、删除（需确认Toast）
- 可返回列表

#### 3.3.4 导入导出

**导出功能**：
- Excel格式导出全部人物
- 导出字段与系统字段一致

**导入功能**：
- 支持上传 Excel 文件（.xlsx 或 .xls 格式）
- 自动解析并导入人物数据
- 提供导入预览和错误提示

**Excel 模板格式**：

| 列名 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名字 | 文本 | 是 | 人物姓名，不可重复 |
| 性别 | 文本 | 是 | 男/女/其他 |
| 年龄 | 数字 | 是 | 单位：岁，范围 0-200 |
| 门派 | 文本 | 否 | 所属门派名称 |
| 门派职位 | 文本 | 否 | 如：掌门、长老、弟子等 |
| 势力 | 文本 | 否 | 如：正道、邪道、中立等 |
| 势力职位 | 文本 | 否 | 如：盟主、堂主、护法等 |
| 兵器 | 文本 | 否 | 武器名称和描述 |
| 武学等级 | 文本 | 是 | 无疆/暮海/阔原/长岭/清岫 |
| Rank | 文本 | 是 | S+/S/S-/A+/A/A-/B+/B/B-/C+/C/C-/D+/D/D- |
| 样貌 | 文本 | 否 | 外貌特征描述 |
| 性格核心 | 文本 | 否 | 性格特点描述 |
| 存在价值 | 文本 | 否 | 人物在故事中的作用和意义 |
| 主要冲突方向 | 文本 | 否 | 人物的核心矛盾和发展方向 |

**Rank 与武学等级对应关系**：

| 武学等级 | 对应 Rank 范围 |
|---------|---------------|
| 无疆 | S+, S, S- |
| 暮海 | A+, A, A- |
| 阔原 | B+, B, B- |
| 长岭 | C+, C, C- |
| 清岫 | D+, D, D- |

**导入流程**：
1. 在人物列表页点击"批量导入"按钮
2. 弹出文件选择对话框，选择 Excel 文件
3. 系统解析文件并显示预览（前 5 条数据）
4. 显示数据验证结果：
   - 成功：显示可导入的记录数
   - 警告：显示格式错误或数据不完整的记录
   - 错误：显示必填字段缺失或数据类型错误的记录
5. 用户确认后执行导入
6. 显示导入结果：成功数、失败数、失败原因

**业务规则**：
- 名字重复的数据将被跳过，并提示用户
- 武学等级与 Rank 不对应的数据将被标记为警告
- 最大支持一次导入 500 条记录

**错误处理**：
| 错误类型 | 处理方式 |
|---------|---------|
| 必填字段为空 | 标记错误，跳过该条记录 |
| 年龄超出范围 | 标记错误，跳过该条记录 |
| 性别值无效 | 标记错误，跳过该条记录 |
| 名字重复 | 标记警告，跳过该条记录 |
| 武学等级与 Rank 不对应 | 标记警告，允许导入但提示用户核对 |

**重名处理**: 自动在重复姓名后加数字后缀（令狐冲→令狐冲1→令狐冲2）

## 第四部分：数据库设计

### 4.1 IndexedDB 结构

```typescript
const db = new Dexie('HabitTrackerApp');

db.version(2).stores({
  // v1.0 习惯数据
  habits: '++id, name, habitType, status, createdAt',
  checkIns: '++id, habitId, date, status',
  
  // v2.0 新增人物数据
  characters: '++id, name, sect, martialLevel, age, createdAt'
});
```

### 4.2 数据隔离
- Habit Tracker 与人物管理系统数据完全隔离
- 各自独立存储，互不影响
- 登录状态存储于 localStorage，与应用数据分离
